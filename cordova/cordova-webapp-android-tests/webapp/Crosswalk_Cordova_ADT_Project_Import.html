Test steps:

  1. Get cordova source code and cd into the folder
    • If check embedded mode project, use command: './bin/create helloworld com.example.helloworld helloworld' to create helloworld project
    • If check shared mode project, use command: './bin/create helloworld com.example.helloworld helloworld --xwalk-shared-library' to create helloworld project
  2. Open ADT by a menu entry to start it, or run the 'eclipse/eclipse' command from inside your Android SDK's root directory
  3. In the File menu, choose Import....
  4. In the Import dialog box, choose Android then Existing Android Code into Workspace. Click the Next button
  5. Choose the directory for the generated project; for example, for the HelloWorld project
  6. Click the Finish button to import the three projects(helloworld, helloworld-CordovaLib, xwalk_core_library)
  7. Wait a minute and then check the error message of the import project
    • If the error massage display 'The import org.apache.cordova cannot be resolved'
    • If the error massage display 'The import com.squareup.okhttp.OkHttpClient cannot be resolved'
  8. Build and Run
    • Select the project, then Debug as > Android Application and check the Logcat message
    • Select the project, then Run as > Android Application and check the Console message
  9. Check the test device
  10. You may need to turn off the automatic build option (uncheck Project > Build Automatically in Eclipse) so you can build the projects manually and in the correct order

Expected Output:

  1. Open the eclipse
  2. Open the menu
  3. Open file browser window
  4. Select the project
  5. The project can import successfully
  6. Analyze the error message and resolve
  7. The project can build successfully and
    • the LogCat display the apk debug message
    • the Console log display the apk install successfully
  8. The project can launch successfully
  9. The project can build and launch successfully
